## Validator — Phases Now Implemented

1. File structure (HEAD first, TRLR last, GEDC.VERS present)
2. Xref format — `@<no-spaces>@`
3. Duplicate xref detection
4. Level stepping (child = parent + 1)
5. Tag legality (parent→child rules from `specification.py`)
6. Cardinality (min/max child counts)
7. Payload type checking (NONE / POINTER / TEXT / ENUM)
8. Payload format — DATE (full grammar incl. INT), TIME, AGE, LATI, LONG, LANG, RESN
9. Enumeration values — SEX, QUAY, MEDI, PEDI, ROLE, RESN
10. Pointer target resolution (dangling pointer detection)
11. `@VOID@` sentinel — allowed only on spec-defined pointer slots
12. Bidirectional links — INDI.FAMC↔FAM.CHIL, INDI.FAMS↔FAM.HUSB/WIFE
13. Orphaned record detection — SOUR/REPO/OBJE/SNOTE defined but never cited
14. TRAN context — TRAN must have LANG; FILE.TRAN must have FORM
15. Extension tag declaration (HEAD.SCHMA.TAG) — bypassed with `--lenient`
16. C0 control characters (NUL = error, other C0 = warning)
17. CONC deprecation warning
18. Line-length estimate (warning if >255 chars)

**Test result on `maximal70.ged`:** 0 errors, 2 warnings (two TRAN without LANG — legitimate issues in that file)

## CLI Tools

### `validat7 [--lenient] <file.ged>`
- Exit 0 = clean, 1 = errors, 2 = not GEDCOM 7, 3 = file not found
- `--lenient` disables strict extension tag checking

### `g7cli [file.ged]`
Interactive shell commands:
```
load <path>                  load a GEDCOM 7 file (warns if unsaved changes)
write <path>                 write to file (clears dirty flag)
validate                     run full validator
info                         file summary with per-tag record counts
ls                           list children of current node
cd <index|tag|@xref@|..>     navigate the tree
pwd                          show current path
show [--all]                 show node fields; --all also runs ls
find <tag> [--payload <txt>] search by tag, optionally filter by payload text
set payload|tag|xref <val>   edit current node in-place
add <tag> [payload]          add child (or top-level record before TRLR)
rm <index>                   remove child by index
help / quit
```
- Dirty flag tracks unsaved edits; `load` and `quit` both prompt before discarding

## Conversion Path Architecture (Plan for Next Session)

```
Gedcom5x  ──parse──►  Element/IndividualRecord/FamilyRecord
                              │
                    gedcom5_to_g7.py (TO BUILD)
                              │
                              ▼
                    List[GedcomStructure]  (GEDCOM 7 tree)
                              │
                         Gedcom7Writer
                              │
                              ▼
                         output.ged  (valid GEDCOM 7)

GedcomX  ──parse──►  GedcomX object graph
                              │
                    gedcomx_to_g7.py (TO BUILD)
                              │
                              ▼
                    List[GedcomStructure]
                              │
                         Gedcom7Writer
```

Key mapping decisions for GEDCOM5→7 converter:
- GEDCOM 5 `CONC` lines are merged by `Gedcom5x` already — writer handles CONT splitting
- `NAME /Surname/ Givename` format → `NAME` + `GIVN`/`SURN` parts
- Date normalization: GEDCOM 5 dates should already be valid g7 DATE grammar
- LDS-specific tags: GEDCOM 7 still supports them
- `OBJE` inline (GEDCOM 5.5) vs record (GEDCOM 5.5.1+) — needs normalization
- `NOTE` inline text → `SNOTE` or inline `NOTE` depending on reuse
- `SUBM` pointer in HEAD is the same in both versions
